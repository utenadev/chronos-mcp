# Chronos MCP: 記憶の永続化と時間自意識

Chronos MCP は、エージェントの会話履歴を「単なるログ」から「長期的な知恵」へと昇華させるための MCP (Model Context Protocol) サーバーです。SQLite をバックエンドに持ち、バージョン管理のような記憶のスナップショット機能と、時間の経過を自覚する「時間自意識」を提供します。

## 主な機能

### 1. 4層メモリ構造 (v2.0)
脳科学の知見に基づき、記憶を以下の4つの階層で管理します。
- **短期記憶 (Simple Memory)**: 会話の生ログ。
- **セマンティック記憶**: ベクトル検索による想起（将来拡張予定）。
- **関連性記憶 (GraphRAG)**: 知識間の因果関係。
- **統合・再編層 (Consolidation Layer) 【新規】**: 睡眠サイクルによる記憶の要約と抽象化。

### 2. 時間自意識フック (v2.1)
エージェントに「時間の流れ」を意識させます。
- **Time Awareness Hook**: ツール実行時に、前回の活動から1時間以上の空白がある場合、自動的に経過時間をコンテキストとして注入します。
- **Ambient Context**: 起動時に天気、記念日、トレンドなどの背景情報を取得し、より自然な対話を可能にします。

### 3. セッション管理
- セッションの開始と終了を明示的に記録し、長期的な思考の変遷を分析可能にします。

## アーキテクチャ

本システムは、高いリアルタイム性が求められるサーバー層と、複雑な推論を行う分析層を分離したハイブリッド構成をとっています。

- **Go (MCP Server / Core)**:
    - MCP ツールの提供（stdio 経由）。
    - SQLite DB への直接的な読み書き。
    - 時間自意識（Time Awareness Hook）によるリアルタイムな文脈注入。
    - 主要なビジネスロジックと高速なデータ永続化。
- **Python (Consolidation Batch / Analyzer)**:
    - 睡眠サイクル（AM 3:00 - 4:00）におけるバックグラウンド処理。
    - LLM を用いた会話ログの要約・抽象化。
    - 記憶間の因果関係（Causal Links）の特定。
    - ペルソナ情報（Persona Anchors）の抽出。

## MCP ツール一覧

- `create_snapshot`: 現在の状態をスナップショットとして保存。
- `checkout_snapshot`: 最新のスナップショットを取得。
- `record_turn`: 会話のターン（発言ペア）を記録。
- `analyze_evolution`: セッション内の思考の変遷を分析。
- `get_ambient_context`: 時間経過や環境情報を含む背景情報を取得。
- `record_session_event`: セッションの開始/終了イベントを記録。
