# chronos-mcp 拡張仕様：時間自意識と情緒的コンテキスト (v2.1)

## 1. 目的
エージェントに「時間の経過」に対する能動的な意識を持たせ、単なる記録ツールから、ユーザーの活動状況や外部環境（天気、トレンド等）を考慮した「気遣いのできるパートナー」へと進化させる。

## 2. 時間経過監視フック (Time Awareness Hook)
全てのMCPツールの実行時に、前回の活動からの経過時間をチェックし、エージェントに「気づき（Observation）」を注入する。

### 仕様
- **実行タイミング**: `ChronosMCPServer.HandleTool` の開始時。
- **ロジック**:
    - 前回のツール実行時刻（またはセッション開始時刻）と現在時刻の差分を計算。
    - 経過時間が特定の閾値（例: 1時間）を超えた場合、以下の情報を生成する。
        - セッション開始からの総経過時間。
        - 進行中の Issue (`beads`) の状況。
        - 他エージェント（OpenCode等）の最終活動時刻からの停滞。
- **出力**: 本来のツール実行結果の末尾に `[Chronos Awareness]` セクションとして情報を付加し、エージェント（LLM）に現状を案じさせる。

## 3. 情緒的コンテキスト取得 (Ambient Context)
起動時に時刻以上の背景情報を提供するが、その扱いは慎重に行う。

### 新ツール：`get_ambient_info`
- **提供情報**:
    - **前回稼働情報**: 「前回は〇〇時間前に、〇〇の作業で終了しました」といった文脈。
    - **外部環境（ランダム）**:
        - 今日の記念日（今日は何の日）。
        - ユーザーの地域の天気情報。
        - 最新の時事ニュースやSNSのトレンド。
- **活用ガイドライン（受動的活用）**:
    - **内部保持**: 取得した情報はエージェントの「背景知識（潜伏記憶）」として静かに保持し、自分から積極的に話し出すことはしない。
    - **発話トリガー**: 
        1. ユーザーが「何か面白い話題ある？」「最近のニュースは？」「外の天気はどう？」など、直接的に情報を求めた場合。
        2. 会話の本筋が取得した情報と密接に関連しており、引用することで対話の質が向上すると判断した場合。
    - **目的**: 常に「背景を知っている」状態を維持することで、ユーザーからの不意の問いかけに即座に、かつ「分かっている」トーンで応じることを目指す。

## 4. セッション管理と終了処理 (`/bye`)
セッションの開始と終了を明示的に管理する。

### 新サブコマンド/ツール：`record_session_event`
- **引数**: `type` ("start" | "end")
- **役割**: DBの `session_logs` テーブルにイベントを記録。
- **Gemini-CLI 連携**: 
    - `/bye` スラッシュコマンド（Skill）実行時に `type="end"` を呼び出す。
    - 終了時に「今日の記憶を整理（海馬リプレイ）しておきますね」といった情緒的な挨拶を行うトリガーとする。

## 5. 追加データモデル (SQLite)

### `session_logs` テーブル
| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | INTEGER | PK |
| `event_type` | TEXT | "start", "end" |
| `timestamp` | DATETIME | 記録時刻 |
| `summary` | TEXT | セッションの簡単なサマリ（終了時） |
