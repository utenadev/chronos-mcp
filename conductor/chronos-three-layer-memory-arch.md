# chronos-mcp 記憶システム：海馬リプレイと統合の設計書 (v2.0)

## 1. 概要
本システムは、脳科学における「海馬リプレイ」と「記憶の統合」のプロセスを模倣し、エージェントの記憶を単なるログの蓄積から、構造化された「知恵」と、ユーザーに対する「深い理解（共感への理解）」へと昇華させることを目的とする。

## 2. 4層メモリ構造
従来の3層構造に、動的な整理プロセスを担う第4の層を追加する。

1.  **シンプルメモリ (短期記憶)**: 会話の生ログ記録。
2.  **セマンティック記憶 (曖昧検索)**: ベクトル検索による想起。
3.  **関連性記憶 (GraphRAG)**: 知識間の因果関係と構造化。
4.  **統合・再編層 (Consolidation Layer)**: **【新規】** 睡眠サイクルによる記憶の整理。

## 3. 統合プロセス：睡眠サイクル (AM 3:00 - 4:00)
専用サーバーにて、エージェントのアイドル時間に以下のバッチ処理を実行する。

### A. 海馬リプレイ (Hippocampal Replay)
その日の「短期記憶」を時系列で再走査し、LLMを用いて以下の要素を抽出する。
- **因果関係の特定**: 「何をした結果、どうなったか」というリンクの生成。
- **抽象化**: 技術的な議論を、再利用可能な「ルール」や「教訓」へ変換。

### B. 夢 (Speculation & Insight)
既存の長期記憶と新しい情報をランダムに突き合わせ、以下の発見を試みる。
- **潜在的な矛盾の検知**: プロジェクト間や過去の発言との不整合の発見。
- **伏線の回収 (Open Loops)**: 過去に「面白そう」と言及されたが未着手のテーマの掘り起こし。

## 4. 重み付けと情報の選別基準
脳が情報を捨てるように、システムも以下の指標に基づいて「記憶の強度」を調整する。

### 1. 実用的再利用性 (Utility)
- 将来のタスク実行やコード生成に役立つ技術的知見。

### 2. ペルソナ・アンカー (Persona Anchors) - 「共感への理解」
- **内容**: ユーザーの嗜好（音楽、マンガ、アニメ）、特有の言い回し、文化的背景。
- **重要性**: 本筋（タスク）以外の「会話のパーツ」を正確に保持することで、ユーザーへの深い理解を示す。
- **保存形式**: 抽象化せず、あえてユーザーの「声」に近い形式（Rawデータ）で長期保存する。

### 3. 思考の変遷 (Evolutionary Drift)
- ユーザーの設計思想や好みが時間の経過とともにどう変化したか。その「変化の軌跡」を記憶の価値とする。

## 5. データ・コントラクト拡張
`search_source` テーブルに、統合プロセスを制御するためのフィールドを追加する。

| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| `is_persona_anchor` | INTEGER | 嗜好や言い回し等のペルソナ情報フラグ |
| `importance_score` | FLOAT | 睡眠プロセスで算出される重要度 (0.0 - 1.0) |
| `causality_id` | TEXT | 因果関係で結ばれた親ノードへのポインタ |
| `status_consolidated` | INTEGER | 統合プロセス完了フラグ |
